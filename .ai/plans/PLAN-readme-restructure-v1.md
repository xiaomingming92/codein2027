## 问题诊断

当前存在的问题：
1. **文件散落**：4 篇哲学理论文章 + 实践.md 散落在项目根目录，没有独立路径
2. **目录碎片化**：`农业智能体(把地种智能体)` 与 `大田精准耕播智能决策系统` 是两个目录但实际是同一系统的早期和当前版本，应合并
3. **三层关系不清晰**：项目实际包含"哲学理论（4篇）→ ADD 范式 → 农业智能体实践"三层金字塔结构，但 README 把范式孤立介绍，没有显式展示三层之间的推导关系
4. **重复/混乱区域**：第 103-107 行有一个合并遗留的断裂段落 `"本项目的 agent 源码是\n# 农业智能体\n\n基于 Next.js..."`，导致结构混乱
5. **实践介绍缺失**：`实践.md` 只有 4 行的表格，没有展开说明"四个缺口"如何通过本项目填补

## 改造目标

1. 将 5 个散落文件归入 `docs/哲学理论/` 独立目录
2. 合并 `农业智能体(把地种智能体)` 到 `大田精准耕播智能决策系统`
3. 将 README 改造为"哲学 → 范式 → 实践"三层金字塔结构
