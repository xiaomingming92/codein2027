# 实践方法论：填补四大缺口

ADD 范式要真正推向行业，面临四个结构性缺口。以下给出每个缺口的典型表现、本项目的实践方案以及验证方式。

---

## 缺口一：缺公共标准

**典型表现**：审计规则、DSL 语义、能力模型各家各写一套，互不兼容，无法形成行业共识。

**本项目的实践方案**：
- 将 ADD 范式的核心概念（裁决层、能力模型、审计阶段、三通道输出）抽象为开源协议/规范，而非私有实现
- 通过 AGPL-3.0 + 强制署名附加条款 建立法律约束，让社区/公司愿意直接引用而不是抄作业
- 在 `project_rules.md` 中固化 ADD-0 到 ADD-7 共 8 条规则，形成可复用的工程规范
- MCP 工具链（`get_project_context`、`check_phase_symmetry` 等）提供可复制的自动化合规检查标准

**验证方式**：任何团队拷贝 `.trae/` 目录即可复现完整的 ADD 开发环境，不需要理解内部实现细节。

---

## 缺口二：缺可验证实现

**典型表现**：文章写得通透，但别人跑不通 Demo；理论自洽，但工程上无法复现。

**本项目的实践方案**：
- 给出"一键启动"仓库 — `git clone` + `npm install` + `npm run dev` 三步跑通
- Podman/Docker 自动检测，跨平台零配置启动 PostgreSQL + ChromaDB
- 内置 Agent 诊断命令 `npm run agent:diagnose`，验证 LLM/ChromaDB/PostgreSQL 连通性
- 审计日志可实时查看：`npm run agent:logs` / `npm run kb:logs`
- 源码完全开放，所有 ADD 范式规则都在 `.trae/` 目录下可审查

**验证方式**：怀疑者在 5 分钟内跑不通 Demo 就算我们输。整个 ADD 开发链路可以在本仓库完整复现。

---

## 缺口三：缺规模化故事

**典型表现**：只能说服技术同学，预算委员会听不懂；技术论证充分，商业论证缺失。

**本项目的实践方案**：把 ROI 简化为三行数字，帮 CTO 在董事会拍桌子：

| 维度 | 传统 AI 编程 | ADD 范式 |
|------|------------|---------|
| 上下文保留 | 单次会话丢失 | 跨轮次/跨对话永久接续 |
| 决策可追溯 | 黑盒 | 全链路白盒审计 |
| 合规成本 | 事后人工补审计 | 编码时自动生成 |

- **人头节约**：AI 自主完成 80% 编码，人类只定义业务目标
- **交付周期**：长周期项目不再被会话中断打断，接续成本趋近于零
- **合规风险**：审计日志自动生成，满足 ISO/SOC2 等审计要求

**验证方式**：本项目自身就是案例 — 3 轮独立对话完成迭代，每轮无需重复复述上下文。

---

## 缺口四：缺组织方法

**典型表现**：一离开原作者就塌房；知识全在人脑里，无法传承。

**本项目的实践方案**：把知识固化到工具链、模板、课程，让新人 1 周能上手：

1. **工具链固化** — MCP 工具（`generate_audit_logger`、`check_phase_symmetry` 等）把范式约束变成可自动检查的规则
2. **SKILL 模板** — `session-init` 和 `add-paradigm` 两个 SKILL 定义了 AI 助手的标准行为模式
3. **onboarding 路径** — 新人只需：
   - Day 1-2：阅读 `docs/哲学理论/` 4 篇文章理解"为什么"
   - Day 3-4：阅读 `project_rules.md` 和 SKILL 文件理解"怎么做"
   - Day 5-7：在本仓库跑通一轮完整 ADD 开发流程
4. **CI 模板** — `.trae/` 目录可复制到任何项目，即时获得 ADD 开发环境

**验证方式**：任何一个理解 ADD 概念的新人，拷贝 `.trae/` 目录到自己的项目后，AI 助手会自动按照 ADD 范式工作。

---

## 总结

| 缺口 | 问题 | 方案 | 一句话 |
|------|------|------|--------|
| 缺公共标准 | 各写一套 | 开源协议 + 工程规范 + MCP 工具链 | 拷贝 .trae/ 即获得标准 |
| 缺可验证实现 | 跑不通 | 一键启动 + 诊断命令 + 全开源 | 5 分钟跑不通算输 |
| 缺规模化故事 | 听不懂 | 人头/预算/交付周期三行 ROI | CTO 能在董事会拍桌子 |
| 缺组织方法 | 人走塌房 | 工具链 + SKILL + onboarding | 新人 1 周上手 |
