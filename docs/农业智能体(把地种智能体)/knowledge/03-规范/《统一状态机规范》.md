# 《统一状态机规范》

## Unified State Machine Specification V1.0

---

# 1. 文档定义

---

## 1.1 文档名称

《统一状态机规范》

---

## 1.2 文档目标

本文档用于定义：

> Agri-Agent OS 平台中所有任务、设备、智能体、作业流程统一使用的状态机模型、状态流转规则与异常控制机制。

本文档属于：

```text
平台级核心规范

```

是农业智能体平台中的：

```text
统一流程控制核心

```
---

## 1.3 文档作用

解决：不同模块状态不一致的问题。

确保：

*   AI
    
*   前端
    
*   后端
    
*   农机
    
*   IoT
    
*   ROS
    

对当前系统状态认知一致。

---

# 2. 状态机总体原则

---

# 2.1 核心原则

所有状态机必须满足：

| 原则 | 说明 |
| --- | --- |
| 唯一状态 | 同一时间只能有一个主状态 |
| 可追踪 | 所有状态变化可记录 |
| 可恢复 | 异常后可恢复 |
| 可中断 | 支持人工终止 |
| 可回退 | 支持失败回退 |
| 实时同步 | 所有端状态一致 |

---

# 2.2 平台统一状态层级

平台统一采用：

```text
任务状态
→ 智能体状态
→ 设备状态
→ 作业状态
→ 子动作状态

```
---

# 2.3 状态同步原则

所有状态变化必须同步至：

| 系统 | 是否同步 |
| --- | --- |
| Web | 是 |
| App | 是 |
| 大屏 | 是 |
| PAD | 是 |
| 日志系统 | 是 |

---

# 3. Task 状态机（核心）

---

# 3.1 定义

Task 是：

```text
平台最核心业务状态单元

```
---

# 3.2 标准状态机

```text
created
→ parsing
→ analyzing
→ planning
→ dispatching
→ executing
→ completed
→ archived

```
---

# 3.3 异常状态

```text
failed
paused
cancelled
timeout

```
---

# 3.4 状态定义

---

## created

任务已创建。

等待AI解析。

---

## parsing

正在进行：

*   NLP解析
    
*   意图识别
    
*   参数提取
    

---

## analyzing

Agri-Brain 正在：

*   农情分析
    
*   风险分析
    
*   知识推理
    

---

## planning

正在生成：

*   处方
    
*   作业计划
    
*   调度计划
    

---

## dispatching

正在：

*   匹配设备
    
*   调度农机
    
*   分配任务
    

---

## executing

设备正在执行。

---

## completed

任务执行成功。

---

## archived

任务归档完成。

---

# 3.5 状态流转规则

---

## 禁止跳状态

禁止：

```text
created
→ executing

```

必须：

```text
完整经过中间流程

```
---

## completed后禁止修改

```text
completed
→ 不允许重新编辑任务

```
---

## archived不可恢复

```text
archived
→ 终态

```
---

# 15. 状态日志规范

---

# 15.1 所有状态变化必须记录

必须记录：

| 内容 | 示例 |
| --- | --- |
| 时间 | timestamp |
| 来源 | Agri-Brain |
| 原状态 | idle |
| 新状态 | analyzing |
| 原因 | 用户触发 |

---

# 15.2 日志结构

```json
{
  "timestamp": 1750000000,
  "source": "agri_brain",
  "from_state": "idle",
  "to_state": "analyzing"
}

```

# 18. 系统边界

---

# 18.1 本规范负责

*   状态定义
    
*   状态流转
    
*   状态同步
    
*   状态日志
    

---

# 18.2 本规范不负责

*   业务逻辑
    
*   AI推理
    
*   电机控制
    
*   底层运动控制
    

---

# 19. 最终定义

统一状态机规范定义为：

> Agri-Agent OS 中所有系统统一使用的流程状态控制规范。

其核心目标是：

```text
让整个农业智能体平台
拥有统一流程认知

```

从而实现：

```text
复杂农业系统的稳定协同运行

```
