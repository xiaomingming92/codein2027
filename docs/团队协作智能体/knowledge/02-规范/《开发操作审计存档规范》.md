# 开发操作审计存档

本文档描述项目中代码改动如何自动存档到数据库，以及后续 AI 助手如何检索历史改动记录。

## 数据流

```
AI 助手修改代码
       │
       ▼
MCP 工具 record_dev_operation
       │  action: "MODIFY" | "CREATE" | "DELETE"
       │  targetType: "API_ROUTE" | "COMPONENT" | "SCHEMA" | "RULE"
       │  targetId: "/api/knowledge/documents"
       │  beforeState: JSON   ← 修改前的关键状态
       │  afterState: JSON    ← 修改后的关键状态
       │  reason: "中文描述"  ← 操作的业务原因
       ▼
prisma.auditLog.create()
       │
       ▼
PostgreSQL (Podman 容器, data/postgres 数据卷)
  ┌───────────────────────────────────────────────────────┐
  │ AuditLog 表                                           │
  │ id         String  @id @default(cuid())               │
  │ userId     String  → User(username: "ai-assistant")   │
  │ action     String  e.g. "MODIFY"                      │
  │ targetType String  e.g. "API_ROUTE"                   │
  │ targetId   String  e.g. "/api/agent/chat/stream"      │
  │ beforeState Json?  操作前的状态快照                    │
  │ afterState  Json?  操作后的状态快照                    │
  │ reason     String? 操作原因（中文）                    │
  │ createdAt  DateTime                                    │
  └───────────────────────────────────────────────────────┘
```

## 写入

### 触发时机

AI 助手每次修改代码后，**必须**调用 MCP 工具 `record_dev_operation`。

### 信息要求

| 字段 | 要求 |
|------|------|
| action | `MODIFY` / `CREATE` / `DELETE` 等动词 |
| targetType | `API_ROUTE` / `COMPONENT` / `SCHEMA` 等分类 |
| targetId | 文件路径或端点路径 |
| beforeState | 操作前的关键状态（JSON 字符串），不能为空 |
| afterState | 操作后的关键状态（JSON 字符串），**对比 before 应清晰可辨** |
| reason | 操作原因的中文描述，说明"为什么做这个改动" |

### 双重保障

MCP Server 启动时自动加载数据库 URL 的优先级：

1. `.trae/mcp.json` 显式注入的 `DATABASE_URL` 环境变量
2. `.trae/scripts/mcp-server.ts` L5 `datasourceUrl` 构造函数参数（代码硬编码兜底）
3. `.env` Prisma 原生读取（创建于 2026-05-14）

### 示例

```
record_dev_operation({
  action: "MODIFY",
  targetType: "API_ROUTE",
  targetId: "/api/agent/chat/stream",
  beforeState: '{ "streaming": "node-level", "thinking": "none" }',
  afterState: '{ "streaming": "3char-subtokens", "thinking": "6-nodes" }',
  reason: "流式聊天改造：节点粒度→3字符子token推送，新增thinking事件"
})
```

### 用户自动创建

`ai-assistant` 用户在首次写入时通过 `connectOrCreate` 自动创建（mcp-server.ts L715-724），无需手动初始化。

## 查询

### 工具

`query_audit_logs` MCP 工具支持多维度检索：

| 参数 | 说明 |
|------|------|
| targetType | 按目标类型精确过滤 |
| action | 按操作类型精确过滤 |
| targetId | 按目标标识精确过滤 |
| keyword | 模糊匹配 action/targetType/targetId/reason |
| sinceMinutes | 时间窗口（分钟前） |

### 跨会话使用

新 AI 会话开始时，应在响应具体需求之前先执行 `query_audit_logs`，结合 `session-init` SKILL 恢复开发上下文。

### 示例

```
query_audit_logs({ targetType: "API_ROUTE" })
  → 所有 API 路由修改记录
query_audit_logs({ keyword: "streaming" })
  → 与 streaming 相关的所有改动
query_audit_logs({})
  → 最近 20 条记录（无条件）
```

## 技术依赖

| 组件 | 用途 |
|------|------|
| PostgreSQL (Podman) | 审计数据持久化，`data/postgres` 数据卷 |
| Prisma ORM | 数据库连接和操作 |
| dotenv | 环境变量加载 |
| `.trae/scripts/mcp-server.ts` | MCP 工具实现 |
| `.trae/mcp.json` | IDE MCP Server 注册与环境变量注入 |
