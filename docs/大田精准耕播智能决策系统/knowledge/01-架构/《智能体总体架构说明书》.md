# 2. 项目总体定义

---

# 2.1 项目定位

Agri-Agent OS 定义为：

> 面向农业生产全过程的农业智能体操作系统。

系统目标：

通过：

*   AI智能体
    
*   农业知识模型
    
*   IoT感知
    
*   自动驾驶
    
*   农机协同
    
*   机器人调度
    

实现农业生产过程数字化、智能化、自主化。

---

# 2.2 项目核心目标

系统不是：

```text
单一农机自动驾驶系统

```

也不是：

```text
普通农业管理软件

```

而是：

> 一个能够理解农业任务、分析农情、生成处方、调度农机、自主完成作业的农业智能化操作系统。

---

# 2.3 系统核心能力

系统需具备：

| 能力 | 说明 |
| --- | --- |
| 任务理解 | 理解用户自然语言任务 |
| 环境感知 | 获取农田实时环境 |
| 农艺推理 | 基于农业知识进行分析 |
| 风险评估 | 判断是否允许执行 |
| 处方生成 | 生成农业作业方案 |
| 能力匹配 | 判断设备是否具备执行能力 |
| 农机调度 | 自动分配执行设备 |
| 自动驾驶 | 自主路径执行 |
| 作业控制 | 控制农机完成作业 |
| 状态反馈 | 实时回传任务状态 |
| 多项目复用 | 支持不同农业场景 |

---

# 3. 系统总体架构

---

# 3.1 系统架构总览

```text
用户
↓
Agri-Agent OS 农业智能体平台
    ↓
    ├── AI智能体核心层
    │
    ├── 农情分析智能体（Agri-Brain）
    │
    ├── 农机执行智能体（Agri-Machine Agent）
    │
    ├── IoT环境感知系统
    │
    ├── 统一能力模型系统
    │
    ├── 风险控制系统
    │
    ├── 数据与知识中心
    │
    └── 多终端交互系统
                ↓
            农机设备层
                ↓
            车载控制系统
                ↓
            执行机构

```
---

# 3.2 系统分层结构

系统分为六层：

| 层级 | 名称 | 职责 |
| --- | --- | --- |
| L1 | 用户交互层 | 用户输入与结果展示 |
| L2 | 平台业务层 | 项目、任务、调度管理 |
| L3 | AI智能体层 | 推理、分析、决策 |
| L4 | 能力与规则层 | 能力约束与执行规则 |
| L5 | 设备执行层 | 农机与机器人执行 |
| L6 | 控制与硬件层 | RTK、控制器、执行机构 |

---

# 4. 核心业务主流程

---

# 4.1 统一业务流程

所有农业场景统一遵循：

```text
任务输入
→ 意图识别
→ 任务结构化
→ 环境感知
→ 农情分析
→ 风险评估
→ 数字处方生成
→ 能力匹配
→ 农机调度
→ 自动驾驶
→ 作业执行
→ 状态反馈
→ 作业归档

```

系统获取：

*   土壤传感器
    
*   气象数据
    
*   RTK位置
    
*   设备状态
    

等环境信息。

---

## 4.2.4 农情分析

系统基于：

*   农业知识库
    
*   作物模型
    
*   环境状态
    

进行农艺推理。

---

## 4.2.5 风险评估

系统判断：

*   是否适宜执行
    
*   是否存在风险
    
*   是否需要人工确认
    

---

## 4.2.6 数字处方生成

生成：

*   播种方案
    
*   施肥方案
    
*   植保方案
    
*   作业参数
    

---

## 4.2.7 能力匹配

判断：

```text
当前设备是否具备完成任务能力

```
---

## 4.2.8 农机调度

自动选择：

*   农机
    
*   机器人
    
*   补给设备
    

执行任务。

## 4.2.9 自动执行

设备完成：

*   作业

## 4.2.10 结果反馈

系统输出：

# 5.1 智能体总体定义

```text
多智能体协同架构
```

# 5.2 智能体分类

*   状态反馈
    

---

## 5.2.3 风险控制智能体

负责：

*   执行门控
    
*   风险校验
    
*   安全确认

## 5.2.4 数据智能体

负责：

*   数据清洗

# 6. AI智能体核心原则

---

# 6.1 AI不直接控制执行机构

系统必须遵循：

```text
AI负责决策
控制器负责执行

```
---

# 禁止：

```text
LLM直接输出：
转向角
油门
液压控制量

```
---

# 正确流程：

```text
AI生成策略
→ 规则引擎校验
→ 控制系统执行

```
---

# 6.2 两阶段执行原则

系统必须分为：

---

## 第一阶段：分析阶段

由：

```text
Agri-Brain

```

负责。

输出：

```text
数字处方

```
---

## 第二阶段：执行阶段

由：

```text
Agri-Machine Agent

```

负责。

---

# 7. 意图控制系统

---

# 7.1 模块目标

控制：

```text
哪些任务允许自动执行

```
---

# 7.2 意图分类

| 类型 | 示例 | 是否允许自动执行 |
| --- | --- | --- |
| 查看类 | 看看葡萄成熟了吗 | 否 |
| 分析类 | 是否缺肥 | 否 |
| 建议类 | 最近做什么农事 | 否 |
| 条件执行类 | 如果有病害就打药 | 可执行 |
| 高风险类 | 收割、喷药 | 需确认 |

---

# 9. 风险控制系统

---

# 9.1 风险分类

| 风险类型 | 示例 |
| --- | --- |
| 环境风险 | 大雨、大风 |
| 作物风险 | 未成熟 |
| 设备风险 | 电量不足 |
| 路径风险 | 无法到达 |
| 执行风险 | 能力不足 |

---

# 9.2 风险结果

| 结果 | 含义 |
| --- | --- |
| allow | 允许执行 |
| deny | 禁止执行 |
| confirm\_required | 人工确认 |
