# 聊天实例管理文档

## 1. 概述

聊天面板支持多会话（Thread）管理，用户可以同时维护多个独立的对话，每个对话拥有独立的消息历史和 AI 上下文。

## 2. 架构设计

### 2.1 整体架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        ChatPanel                                 │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │ ChatThreadTab (会话管理)                                     ││
│  │ [+] [对话1: 水稻方案] [对话2: 病虫害] [对话3: ...]           ││
│  └─────────────────────────────────────────────────────────────┘│
│  ┌─────────────────────────────────────────────────────────────┐│
│  │ 头部: 智能助手 [就绪/思考中]                                  ││
│  └─────────────────────────────────────────────────────────────┘│
│  ┌─────────────────────────────────────────────────────────────┐│
│  │ ChatContainer (消息列表 + 输入框)                             ││
│  │   ← 显示当前活跃 Thread 的消息                               ││
│  └─────────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────────┘
         │
         │ fetch POST /api/agent/chat { threadId, messages }
         ▼
┌─────────────────────────────────────────────────────────────────┐
│ API Layer                                                        │
│  threadId = tid || randomUUID()                                  │
│  runAgent(input, { configurable: { thread_id: threadId } })      │
└─────────────────────────────────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────────────────────────────────┐
│ LangGraph Agent                                                  │
│  MemorySaver 按 thread_id 持久化对话状态                          │
│  intention → retrieval → reasoning → verdict → response          │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 数据模型

```typescript
interface Thread {
  id: string              // 会话唯一标识，格式: thread-{timestamp}-{random}
  title: string           // 会话标题，默认 "新对话"，首条消息后自动更新
  messages: Message[]     // 该会话的消息列表
  createdAt: string       // 创建时间 (ISO)
  updatedAt: string       // 最后更新时间 (ISO)
  isStreaming: boolean    // 该会话是否正在等待 AI 响应
}

interface Message {
  role: "user" | "assistant" | "system"
  content: string
  name?: string
  timestamp?: string
}
```

### 2.3 状态管理

状态由 Zustand store (`chat-store.ts`) 管理：

| 状态 | 类型 | 说明 |
|------|------|------|
| `threads` | `Thread[]` | 所有会话列表 |
| `activeThreadId` | `string \| null` | 当前活跃会话 ID |
| `isStreaming` | `boolean` | 全局流式状态（向后兼容） |

**核心方法**：

| 方法 | 说明 | 线程安全 |
|------|------|---------|
| `createThread()` | 创建新会话，返回 threadId | ✅ |
| `switchThread(threadId)` | 切换活跃会话 | ✅ |
| `deleteThread(threadId)` | 删除会话，自动切换 | ✅ |
| `updateThreadTitle(threadId, title)` | 更新会话标题 | ✅ |
| `addMessageToThread(threadId, message)` | 向指定会话添加消息 | ✅ 关键！ |
| `setStreamingForThread(threadId, streaming)` | 设置指定会话的流式状态 | ✅ 关键！ |

**Selector hooks**：

| Hook | 说明 |
|------|------|
| `useActiveMessages()` | 返回当前活跃会话的消息列表 |
| `useActiveThreadId()` | 返回当前活跃会话 ID |
| `useActiveThreadStreaming()` | 返回当前活跃会话的流式状态 |

## 3. 竞态条件防护

### 3.1 问题描述

用户在会话 A 发送消息后，AI 响应返回前切换到会话 B。如果 `addMessage` 写入的是 `activeThreadId`，则 AI 响应会被错误地写入会话 B。

### 3.2 解决方案

**核心原则：消息始终写入发起请求时的 threadId，而非当前活跃的 threadId。**

```
handleSend() {
  targetThreadId = activeThreadId  // 闭包捕获

  addMessageToThread(targetThreadId, userMessage)  // 写入发起时的线程
  setStreamingForThread(targetThreadId, true)       // 设置发起线程的流式状态

  fetch("/api/agent/chat", { threadId: targetThreadId })
    .then(data => {
      addMessageToThread(targetThreadId, assistantMessage)  // 写入发起时的线程
    })
    .finally(() => {
      setStreamingForThread(targetThreadId, false)           // 清除发起线程的流式状态
    })
}
```

**额外防护：AbortController**

每个会话维护独立的 `AbortController`，同一会话重复发送时自动取消前一次请求：

```
abortControllers = Map<threadId, AbortController>

handleSend() {
  existingController?.abort()  // 取消同线程的旧请求
  controller = new AbortController()
  abortControllers.set(targetThreadId, controller)

  fetch(url, { signal: controller.signal })
}
```

## 4. 交互操作

### 4.1 新建对话

- 点击 **[+]** 按钮
- 自动生成 `threadId`，创建空会话
- 切换到新会话

### 4.2 切换对话

- 点击 Tab 标题切换
- 消息列表自动切换到对应会话
- 正在进行的 AI 响应不受影响（后台继续，写入原线程）

### 4.3 删除对话

- 点击 Tab 右侧 **[×]** 按钮
- 删除当前会话 → 自动切换到相邻会话
- 删除最后一个会话 → 自动创建新对话

### 4.4 重命名对话

- **双击** Tab 标题进入编辑模式
- 输入新标题（最大 50 字符）
- **Enter** 或 **失焦** 保存
- **Escape** 取消编辑

### 4.5 自动命名

- 首条用户消息发送后，自动提取前 20 字作为 Tab 标题
- 后续消息不覆盖已修改的标题

## 5. API 接口

### 5.1 请求

```typescript
POST /api/agent/chat
{
  messages: Array<{ role: string; content: string }>,
  threadId?: string  // 会话 ID，不传则自动生成
}
```

### 5.2 响应

```typescript
{
  success: boolean,
  data: {
    messages: Array<{ role: string; content: string }>,
    verdict?: { ... },
    currentTask?: { ... },
    threadId: string  // 服务端使用的 threadId（新生成或复用）
  }
}
```

### 5.3 LangGraph 对话持久化

`threadId` 传递给 LangGraph 的 `MemorySaver` checkpointer：

```typescript
runAgent(input, { configurable: { thread_id: threadId } })
```

- 相同 `threadId` → LangGraph 恢复之前的对话状态
- 不同 `threadId` → 全新的对话上下文
- `MemorySaver` 是内存级持久化，服务重启后丢失

## 8. 故障排查

### 8.1 消息写入错误线程

**症状**：在会话 A 发送消息，AI 响应出现在会话 B

**原因**：`addMessage` 使用 `activeThreadId` 而非发起时的 threadId

**排查**：
```bash
grep "CHAT_REQUEST\|CHAT_RESPONSE" logs/agent/agent-audit.log | tail -20
```

**验证**：确认 `chat-panel.tsx` 使用 `addMessageToThread(targetThreadId, ...)` 而非 `addMessage()`

### 8.2 切换会话后状态未更新

**症状**：切换 Tab 后消息列表不变

**原因**：`useActiveMessages()` selector 未正确订阅

**排查**：检查 `ChatContainer` 的 `messages` prop 是否来自 `useActiveMessages()`

### 8.3 thread_id 缺失导致 500

**症状**：`Failed to put checkpoint. The passed RunnableConfig is missing a required "thread_id"`

**原因**：前端未传 `threadId`，API 层未自动生成

**排查**：
```bash
grep "thread_id" logs/agent/agent-audit.log
```

**验证**：确认 `chat/route.ts` 有 `threadId = tid || randomUUID()`
