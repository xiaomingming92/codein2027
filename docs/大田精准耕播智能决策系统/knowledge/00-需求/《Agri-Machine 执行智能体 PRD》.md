# 《Agri-Machine 执行智能体 PRD》

## Agri-Machine Execution Agent PRD V1.0

---

# 1. 文档定义

---

## 1.1 文档名称

《Agri-Machine 执行智能体 PRD》

---

## 1.2 文档目标

本文档用于定义：

> Agri-Agent OS 中“农机执行智能体（Agri-Machine Agent）”的整体业务逻辑、设备调度逻辑、自动驾驶流程、任务执行机制、设备状态机与作业控制规则。

本文档属于：

```text
二级核心PRD

```

是农业智能体体系中的：

```text
农业执行中枢

```
---

## 1.3 系统定位

Agri-Machine 不是：

```text
单纯的自动驾驶系统

```

也不是：

```text
普通农机管理平台

```

Agri-Machine 定义为：

> 一个能够接收农业任务、调度农业设备、控制农机作业、管理机器人协同并形成作业闭环的农业执行智能体。

---

# 4. 执行智能体总体流程

---

# 4.1 标准执行流程

所有农业任务统一执行流程：

```text
接收任务
→ 校验能力
→ 匹配设备
→ 生成执行路径
→ 调度设备
→ 前往目标区域
→ 执行作业
→ 返回停机坪
→ 完成归档

```
---

# 4.2 标准执行原则

系统必须遵循：

```text
所有任务执行完成后
设备必须返回停机坪

```
---

# 20. 最终定义

Agri-Machine 定义为：

> Agri-Agent OS 中负责农业设备调度、农业任务执行、自动驾驶流程管理与作业闭环控制的农业执行智能体。

其核心目标是：

```text
让农业设备具备自主执行能力

```

而不是：

```text
仅仅远程控制农机

```
