# 农机智能体产品需求文档（PRD）

---

## 一、产品定义与目标

### 1.1 产品最终目标

让耕播机在给定田块内，基于北斗高精定位、农艺参数和地块环境数据，自主完成路径规划、直线播种、边界识别、U型掉头、播肥参数控制和异常处置，实现高质量、低重播、低漏播的标准化耕播作业。

### 1.2 核心理念

农机智能体需要同时具备**三个大脑**：

| 大脑类型 | 功能描述 |
|---------|---------|
| **行驶大脑** | 让机器会走、走得直、会掉头、不漏播不重播 |
| **作业大脑** | 让机器知道什么时候该播、播多少、播多深、配多少肥 |
| **安全大脑** | 让机器知道什么时候必须减速、停车、人工接管 |

### 1.3 开发原则

采用**"先自动驾驶，再智能作业，最后自主决策"** 三阶段推进，不一上来就追求完全无人。

## 二、核心能力体系（6大能力包）

### 能力包1：高精定位与自动驾驶能力

**目标**
- 北斗高精定位
- 自动循迹直线行驶
- 按规划路线整齐播种
- 地头自动U型掉头
- 自动进入下一条作业带

**需要的能力**
- RTK级北斗定位
- IMU姿态融合
- 方向盘/转向阀自动控制
- 车速闭环控制
- 作业轨迹生成与跟踪控制
- 地头识别与转弯控制

**建议指标**
- 直线作业横向误差：≤2.5cm
- 掉头后入行误差：≤5cm
- 漏播率、重播率：尽量控制在2%以下
- 自动作业覆盖率：一期先做到80%以上

---

### 能力包2：田块边界识别与地头作业能力

**多源融合方案**
- **电子地块边界**：平台提前建档，导入田块多边形
- **高精定位**：实时判断是否接近边界
- **视觉/激光辅助**：识别沟渠、田埂、障碍、地头空旷区域
- **作业机具状态**：判断当前是否适合抬机、减速、转向

**建议做法**

先做：**地块电子围栏 + 北斗定位 + 地头规则控制 + 视觉辅助防碰撞**，更稳、更容易落地。

**U型掉头逻辑**

智能体要能根据以下参数自动计算：
- 机身长度
- 机具宽度
- 最小转弯半径
- 地头预留宽度
- 当前速度
- 土壤打滑情况

自动计算：
- 提前多少米减速
- 何时停播/抬机
- 何时开始转向
- 何时进入下一作业带
- 何时恢复播种/施肥

---

### 能力包3：智能播种作业控制能力

**智能体要控制的关键参数**
- 行驶速度
- 播种量
- 肥料施用量
- 播种深度
- 行距/带宽
- 作业启停
- 地头补偿
- 漏播/重播补偿

**控制思路**

做一个**播种作业控制器**，把"农艺处方"翻译成"执行命令"。

示例逻辑：
- 土壤偏干 → 适当加深播种深度
- 土壤肥力较低 → 局部增加底肥量
- 墒情较好 → 可适当提升作业速度
- 地块边角区域 → 降速并精细控制播种启停

**一期建议**

先做**分区处方 + 固定规则控制**，不要一开始就完全依赖大模型。播种控制需要强确定性，前期要以规则引擎为主、AI为辅。

---

### 能力包4：农艺决策能力（自动生成播种方案）

**输入数据**
| 数据类型 | 具体内容 |
|---------|---------|
| 地块基础信息 | 面积、形状、长度方向、坡度 |
| 土壤数据 | 土壤墒情、土壤肥力 |
| 历史数据 | 历史产量 |
| 作物信息 | 作物类型与品种、播期 |
| 环境数据 | 天气预报 |
| 机具参数 | 机具参数 |
| 目标数据 | 目标亩穗数/基本苗/产量目标 |

**输出结果（机器可执行的播种处方方案）**
- 推荐行驶速度
- 推荐播种量（kg/亩）
- 推荐肥料量（kg/亩）
- 推荐播种深度（cm）
- 推荐作业方向
- 推荐作业顺序
- 地头预留宽度
- 特殊区域差异化策略

**决策引擎：双层决策**

**第一层：规则引擎**（把成熟农艺经验固化）
- 小麦/水稻不同模式
- 不同墒情对应播深
- 不同肥力对应施肥量
- 不同地形对应作业速度

**第二层：AI优化引擎**（基于历史作业数据持续优化）
- 哪种速度下播种均匀度更高
- 哪种深度下出苗更稳定
- 哪种转弯策略减少漏播重播
- 哪种处方在类似田块更高产

**核心原则：先用规则保证能用，再用AI不断变聪明。**

---

### 能力包5：安全与接管能力

**必备安全机制**
- 障碍物检测
- 人员闯入检测
- 紧急停车
- 失去RTK信号自动降级
- 打滑异常告警
- 播种机构堵塞告警
- 肥箱/种箱余量告警
- 超出地块边界自动停车
- 人工一键接管

**建议安全等级**

一期定义为：**L2-L3级农机智能辅助作业**
- 可自动直线作业
- 可自动掉头
- 驾驶员/安全员在场监护
- 支持人工接管

---

### 能力包6：数字孪生与远程运营能力

**平台功能**
- 地块管理
- 农机档案
- 任务下发
- 处方生成
- 实时位置监控
- 作业质量分析
- 作业回放
- 异常告警
- 数据沉淀与模型训练

**长期价值**

平台会成为扩展到耕整机、插秧机、植保机、收割机的**统一底座**。

## 三、技术架构（四层结构）

### 第一层：云端智能决策平台

负责"算"。

**主要模块**
- 地块数字档案
- 土壤与墒情数据管理
- 播种处方引擎
- 路径规划引擎
- 农艺知识库
- 作业任务中心
- 远程监控中心
- 模型训练与迭代中心

---

### 第二层：车载智能体控制器

负责"想"和"指挥"。

**主要模块**
- 实时定位融合模块
- 自动驾驶控制模块
- 路径跟踪模块
- U型掉头模块
- 播肥作业控制模块
- 安全监控模块
- 人机交互模块
- 本地边缘计算模块

这是整台机器的大脑。

---

### 第三层：感知系统

负责"看"和"感知"。

**必选传感器**
- 北斗RTK
- IMU
- 轮速传感器
- 转角传感器
- 液压/电机反馈
- 种箱余量传感器
- 肥箱余量传感器

**强烈建议**
- 前视摄像头
- 广角摄像头
- 毫米波雷达或激光雷达（至少做近距防碰撞）
- 作业深度传感器
- 播种轴转速传感器
- 堵种/堵肥检测传感器
- 土壤墒情传感器（可分在线/离线两种）

**二期可扩展**
- 机具振动监测
- 滑移率估算
- 土壤阻力监测
- 视觉行间质量检测

---

### 第四层：执行机构

负责"动"。

**包括**
- 转向执行器
- 油门/速度控制器
- 制动控制
- 播种电控单元
- 施肥电控单元
- 开沟/覆土/镇压联动控制
- 抬机/落机控制

---

## 四、开发阶段规划

### 阶段1：自动驾驶MVP

**目标**：先解决"机器能自己整齐地播"

**增加能力**
- 直线循迹
- 自动作业路径生成
- 地头自动掉头
- 启停播控制
- 作业轨迹记录

**此阶段不追求**
- 自动生成最佳农艺方案
- 复杂视觉边界理解
- 全无人

**阶段1产出**：一个"能在标准田块稳定工作的自动耕播机"

---

### 阶段2：智能播肥控制

**目标**：再解决"播得对"

**增加能力**
- 速度-播量联动
- 播深可控
- 施肥量可控
- 地块分区参数配置
- 漏播/重播补偿逻辑
- 机具工况监测

**阶段2产出**：一个"能按处方精准作业"的智能耕播机

---

### 阶段3：农艺决策智能体

**目标**：再解决"自己会算"

**增加能力**
- 根据墒情、肥力、天气自动生成方案
- 不同地块差异化参数推荐
- 作业前方案模拟
- 作业后质量评估与回灌学习

**阶段3产出**：一个"会自主决策的耕播智能体"

---

### 阶段4：多机协同与无人值守

**目标**：最后再解决"规模化"

**增加能力**
- 多台农机协同作业
- 云端调度
- 一个安全员看多台机器
- 自动充能/补给衔接
- 全流程作业闭环

**阶段4产出**：一个"可规模化运营的无人农场系统"

## 五、播种方案引擎设计

### 1）处方变量体系

处方最少包含以下字段：
| 字段 | 说明 |
|-----|------|
| 地块ID | 唯一标识 |
| 作物 | 作物类型 |
| 品种 | 作物品种 |
| 播期 | 播种时间 |
| 土壤墒情等级 | 不足/适宜/过多 |
| 土壤肥力等级 | 低/中/高 |
| 推荐作业方向 | 优先长边/短边 |
| 推荐速度 | km/h |
| 推荐播种量 | kg/亩 |
| 推荐施肥量 | kg/亩 |
| 推荐播深 | cm |
| 地头预留距离 | m |
| 转弯模式 | U型/半U型/分段 |
| 特殊区域策略 | 边缘处理方式 |

---

### 2）规则库示例（小麦场景）

- 墒情不足 → 播深增加
- 土壤黏重 → 速度降低
- 肥力偏低 → 底肥增加
- 晚播 → 播量适当增加
- 地块狭长 → 优先长边方向作业
- 地头宽度不足 → 切换半U或分段掉头策略

---

### 3）机器学习引入时机

等你积累了这些数据再上AI：
- 地块条件
- 机械参数
- 实际速度
- 实际播量
- 实际播深
- 出苗情况
- 产量结果

然后训练：
- 最优作业速度模型
- 最优播深模型
- 产量响应模型
- 掉头效率模型

## 七、关键难点分析

### 难点1：机具控制建模难

耕播机不是普通车辆，它后面挂着复杂机具。转向、车速、耕深、播量、覆土、镇压之间会相互影响。

---

### 难点2：田间环境不稳定

- 打滑
- 土质变化
- 地头不规则
- 沟渠障碍
- RTK遮挡

都会影响控制精度。

---

### 难点3：播种质量很难只靠位置精度保证

直线走得整齐，不代表播得好。真正难的是：
- 不漏播
- 不重播
- 深浅一致
- 种肥协同稳定

---

### 难点4：边界识别不能只靠视觉

田块边缘可能模糊、反光、泥泞、杂草遮挡。必须用**电子围栏+高精定位+视觉辅助**。

---

### 难点5：安全责任必须提前设计

农机不同于室内机器人，一旦失控，风险很高。所以安全机制必须前置。

## 十、典型应用场景

### 场景一：耕播一体无人作业

一台农机完成：深耕、播种、施肥、镇压、开沟

实现**一次下地完成多工序作业**

多台农机自动协同：
- 自动分配作业区域
- 自动规划路径
- 自动避让

当种子或肥料不足时：
- 系统自动提醒
- 调度补给车
- 自动补料

### 场景四：无人农场

农机 + 传感器 + AI 形成**全自动农业生产系统**
