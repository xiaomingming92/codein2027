# Agri-Brain Agricultural Intelligence PRD V1.0

---

# 1. 文档定义

---

## 1.1 文档名称

《Agri-Brain 农情分析智能体 PRD》

---

## 1.2 文档目标

本文档用于定义：

> Agri-Agent OS 中“农情分析智能体（Agri-Brain）”的整体业务逻辑、AI分析流程、农业知识推理机制、风险评估机制以及数字处方生成机制。

本文档属于：

```text
二级核心PRD

```

是农业智能体体系中的：

```text
农业认知大脑

```
---

## 1.3 系统定位

Agri-Brain 不是：

```text
普通农业管理系统

```

也不是：

```text
聊天问答机器人

```

Agri-Brain 定义为：

> 一个能够理解农业环境、分析农情状态、调用农业知识、生成农业处方与农事建议的农业认知智能体。

---

# 2.1 总体目标

实现：

```text

转向AI推理驱动
```

系统需具备：

| 能力 | 说明 |
| --- | --- |
| 环境感知 | 获取农田实时状态 |

# 7. 农艺推理引擎

---

# 7.1 模块目标

根据：

*   环境状态
    
*   作物状态
    
*   农业知识
    

进行：

```text
农业逻辑推理

```
---

# 7.2 推理流程

```text
环境分析
→ 作物状态分析
→ 知识匹配
→ 风险判断
→ 农艺决策

```
---

# 7.3 推理类型

| 类型 | 示例 |
| --- | --- |
| 成熟推理 | 是否达到采摘条件 |
| 病害推理 | 是否存在病害风险 |
| 墒情推理 | 是否适宜播种 |
| 气象推理 | 是否适合作业 |
| 物候推理 | 当前生育阶段 |

---

# 7.4 推理规则示例

---

## 葡萄成熟判断

```text
颜色 = 紫色
→ 判定成熟

```
---

## 水稻收割判断

```text
成熟度不足
→ 禁止收割

```
---

## 喷药风险判断

```text
风速过大
→ 禁止喷药

```

# 8. 风险评估系统

---

# 8.1 模块目标

判断：

```text
任务是否允许执行

```
---

# 8.2 风险分类

| 风险 | 示例 |
| --- | --- |
| 环境风险 | 大风、大雨 |
| 作物风险 | 未成熟 |
| 操作风险 | 高风险喷药 |
| 设备风险 | 农机异常 |
| 路径风险 | 无法到达 |

---

# 8.3 风险等级

| 等级 | 说明 |
| --- | --- |
| low | 低风险 |
| medium | 中风险 |
| high | 高风险 |

---

# 8.4 风险结果

```json
{
  "risk_result": {
    "status": "deny",
    "reason": "水稻未成熟"
  }
}

```
---

# 8.5 风险控制原则

系统必须遵循：

---

## 未成熟禁止收割

```text
未成熟
→ 禁止收割

```
---

## 无目标禁止调度

```text
没有成熟葡萄
→ 不调度采摘机器人

```
---

## 高风险作业需确认

```text
高风险喷药
→ 人工确认

```

# 13. 标准业务场景

---

# 13.1 葡萄采摘场景

用户输入：

```text
帮我看看葡萄成熟了吗，
成熟的帮我采摘。

```

系统流程：

```text
识别成熟目标
→ 判断数量
→ 生成采摘处方
→ 输出采摘任务

```
---

# 13.2 无成熟目标场景

系统发现：

```text
没有成熟葡萄

```

系统输出：

```text
当前没有成熟葡萄，
未调度采摘机器人。

```
---

```text
帮我收割水稻。

```

系统输出：

```text
当前水稻未成熟，
禁止收割。

```

# 13.4 病害植保场景

# 16. 最终定义

Agri-Brain 定义为：

> Agri-Agent OS 中负责农业认知、农艺推理、风险分析与数字处方生成的农业认知智能体。

其核心作用是：

```text
让AI真正理解农业

```

而不是：

```text
仅仅管理农业数据

```
